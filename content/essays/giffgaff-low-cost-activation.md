# giffgaff：一次低成本激活与保号的记录

这不是一份“买得越多越好”的攻略。我的目标很简单：让卡顺利激活、确认号码可用，然后用尽量少的操作维持它的状态。

> 运营商规则、资费和漫游政策会变动。下单与充值前，还是以 giffgaff 当期官网说明为准。

## 先把目标拆开

我把这件事分成三段：

1. **激活**：SIM 卡被系统识别，并完成首次开通；
2. **验证**：能收验证码、能发出一条短信或完成一次通话；
3. **保号**：在规则允许的间隔内留下有效使用记录。

这样做的好处是，不会把“首次充值”“买月包”和“长期维持号码”混成一件事。

## 激活当天

卡到手后，我先确认卡片编号、使用的手机是否解锁，以及是否能连接网络。随后在官网的 activation 页面输入 SIM 卡号，按提示完成账户设置。

激活完成后不要立刻收起来。最稳妥的检查顺序是：

- 看手机是否出现运营商名称；
- 收一条短信验证码；
- 发一条短信给可信联系人，或进行一次短通话；
- 登录账户确认号码与余额显示正常。

这几步并不花多少时间，却能避免日后才发现“卡已插过，但实际上没有完成开通”的情况。

## 低成本的思路

如果只是接收验证码或保留号码，我不会把月包当成默认选项。先看当期最低的单次充值和有效期规则，再决定是否需要 data plan。

我的原则是：

> 先满足“有效使用”的条件，再购买真正用得上的流量。

平时把余额、上一次使用日期、下一次提醒日记在日历里即可。提醒不必设得太密；重要的是留出缓冲，而不是等到最后一天才想起。

## 一个小清单

| 事情     | 完成后确认什么         |
| -------- | ---------------------- |
| 激活     | 账户中能看到号码       |
| 首次使用 | 短信或通话有记录       |
| 余额     | 知道最低补充成本       |
| 提醒     | 下次检查日期已写入日历 |

号码对我而言更像一把备用钥匙：平时安静放着，需要时能打开门就好。把流程做简单，反而更容易长期坚持。
